## P0-1: Admin Integrations — API Keys Can't Be Saved

**Root cause:** The `/admin/integrations` page rendered a static form with no save handler and no backend.

**Fix:**
1. Added `org_settings` table to `database/schema.sql` — key/value store scoped by `org_id`, with UPSERT semantics.
2. Ran `npm run db:migrate` — table confirmed created.
3. Created `app/api/admin/integrations/route.ts`:
   - `GET` — returns `{ KEY: boolean }` for each of 5 allowed integration keys, reflecting both env vars and DB values.
   - `POST` — accepts `{ key, value }`, validates against allowlist, upserts into `org_settings`.
4. Replaced `AdminIntegrationsView` in `components/pages/admin.tsx` to be a real server component that loads saved state from DB and passes it to the new `AdminIntegrationsClient`.
5. Added `AdminIntegrationsClient` to `components/pages/workbench-clients.tsx` — per-integration save/update flow with async fetch, saved-state indicator, and inline error display.

**Verification:**
```bash
# Save a key
curl -X POST http://localhost:3000/api/admin/integrations \
  -H "Content-Type: application/json" \
  -d '{"key":"SENDGRID_API_KEY","value":"SG.test123"}'
# → {"ok":true,"key":"SENDGRID_API_KEY","saved":true}  HTTP 200

# Confirm reflected
curl http://localhost:3000/api/admin/integrations
# → {"SENDGRID_API_KEY":true,"TELNYX_API_KEY":false,...}  HTTP 200

# Invalid key
curl -X POST http://localhost:3000/api/admin/integrations \
  -d '{"key":"UNKNOWN","value":"x"}' -H "Content-Type: application/json"
# → {"error":"key must be one of: ..."}  HTTP 400
```

**Files changed:**
- `database/schema.sql` — added `org_settings` table + index
- `types/models.ts` — not updated (no runtime model needed for kv settings)
- `app/api/admin/integrations/route.ts` — NEW file (+49 lines)

## P0-3: POST /api/bids Crashes 500 on Missing Title

**Root cause:** `createBid` was called directly with unvalidated input; missing `title` triggered a Postgres NOT NULL constraint violation (500).

**Fix:** Added early validation in `app/api/bids/route.ts` POST handler before `createBid` is ever called:
- `title` missing or blank → 400
- `line` not "framed"/"frameless" → 400
- `price_margin` missing or not a finite number → 400
- `lines` not an array or empty → 400

**File changed:**
- `app/api/bids/route.ts` (+14/-5 lines)

**Verification:**
```bash
# Missing title
curl -X POST http://localhost:3000/api/bids \
  -d '{"line":"framed","price_margin":0.23,"lines":[{"description":"t","qty":1,"list_cents":100}]}' \
  -H "Content-Type: application/json"
# → {"error":"title is required"}  HTTP 400

# Blank title
curl -X POST http://localhost:3000/api/bids \
  -d '{"title":"  ","line":"framed","price_margin":0.23,"lines":[...]}' -H "..."
# → {"error":"title is required"}  HTTP 400

# Empty lines array
curl -X POST http://localhost:3000/api/bids \
  -d '{"title":"Test","line":"framed","price_margin":0.23,"lines":[]}' -H "..."
# → {"error":"lines must be a non-empty array"}  HTTP 400
```
